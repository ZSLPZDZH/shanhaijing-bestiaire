# Desktop Beast Detail Design

## Goal

让电脑用户在详情弹窗中一次看清完整神兽插图，同时保留原文、要点和通俗解释的连续阅读体验；手机端现有布局与图片呈现方式保持不变。

## Problem

详情弹窗当前最大宽度为 `700px`，图片区域固定为 `300px` 高，并对竖版插图使用 `object-fit: cover`。仓库内插图采用约 `0.73:1` 的竖版比例，因此桌面端会把完整插图强制裁成横幅，神兽的头部、四肢和背景经常超出可视区域。

问题来自容器比例与原图比例不匹配，不是图片资源缺失，也不能通过继续增大横幅高度稳定解决。

## Recommended Layout

桌面端采用两级响应式策略：

1. 宽度为 `769px` 至 `979px` 时保持上下布局，但扩大弹窗，并将详情图改为 `object-fit: contain`，确保整张插图可见。
2. 宽度达到 `980px` 时改为双栏弹窗：左侧固定展示完整插图，右侧承载文字信息并独立滚动。
3. 宽度不超过 `768px` 时继续使用现有手机布局和图片铺满方式，不改变已经可用的手机体验。

宽屏双栏中，弹窗宽度上限约为 `1120px`，高度受视口限制。图片区域使用暗色背景承接原图比例产生的少量留边；文字区域独立滚动，因此用户阅读长内容时，神兽全图会一直留在视野中。

## Interaction

- 打开、关闭、点击遮罩关闭和 `Escape` 关闭行为保持不变。
- 关闭按钮继续固定在弹窗右上角，不随文字滚动消失。
- 桌面端只滚动右侧文字区域，避免整张图片随长文被卷走。
- 中等宽度电脑继续滚动整个弹窗，避免双栏内容过窄。
- 无图片条目继续显示现有占位符。

## Visual Rules

- 完整显示优先于铺满区域，桌面详情图不再裁切。
- 图片不放大超过容器边界，不改变原图纵横比。
- 留边使用项目现有深色背景和细微渐变，不新增装饰元素。
- 双栏之间使用现有边框色分隔，保持当前《山海经》视觉风格。
- 不修改列表卡片缩略图；缩略图仍承担快速浏览用途，完整观看由详情弹窗负责。

## Implementation Boundary

改动集中在 `index.html` 的详情弹窗 CSS，必要时只补充无障碍属性；不修改数据结构、图片资源、搜索筛选逻辑或部署方式，不新增依赖。

## Verification

使用本地静态服务器和浏览器验证：

- 在 `1024×768`、`1440×900` 和 `1920×1080` 视口中打开“鹿蜀”等代表性条目，神兽头部、四肢和完整画面均可见；
- 宽屏下右侧长文可独立滚动，左侧图片保持可见；
- 在约 `800px` 宽的中等视口中，完整图片可见且弹窗可继续向下滚动；
- 在 `390×844` 手机视口中，布局和现有图片铺满效果没有回归；
- 关闭按钮、遮罩点击和 `Escape` 均可正常关闭弹窗；
- 页面控制台没有资源 404 或 JavaScript 错误。

## Out of Scope

- 重做卡片网格或首页视觉；
- 新增图片缩放、下载或全屏查看功能；
- 修改手机端详情布局；
- 重绘、裁剪或替换任何神兽图片；
- 自动推送分支或部署生产站点。
